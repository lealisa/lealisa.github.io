Redesigning the information architecture of Gaia to support advanced layer interactions

  • Role

    Design lead on the project

  • Responsibilities

    Research, Strategy, UX/UI, Prototyping, Visual Design & Front-end Support

  • January–September 2019

Palantir Gaia is a geospatial operations and analysis application that enables users to collaboratively gather intelligence and plan missions to assist warfighters and first responders around the globe.

This project was a cross-team effort to reimagine how layers in the application are built from the back-end, to how they are visualized on the front-end. The goal was to transform layers into first-class objects, in order to make them editable, to allow them to retain data and authorship provenance, and to enable them to dynamically update. By implementing these fundamental architectural changes we could exponentially increase user efficiency, empower more meaningful collaboration, and pave the way for more granular security.

Since layers are so integral to Gaia’s funcionality, this project touched on every part of the application, resulting in a major redesign of the entire system’s information architecture and user interface.
