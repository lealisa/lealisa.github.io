Unifying entry points across all applications in Foundry

  • Role

    Design lead on the project

  • Responsibilities

    UX/UI, Prototyping, Visual Design & Illustration

  • December 2017

Palantir Foundry is a powerful data platform that is backed by an expanding suite of applications for data integration, analysis, visualization, security, etc. However, because applications are managed by different internal teams, there were a lot of inconsistencies between their entry points and onboarding.

This was a self-initiated project that aimed to improve and unify zero states across all apps in Foundry, as well as facilitate discoverability and forge a stronger brand presence.

Context

While working on Contour, I noticed that the zero state did not provide a lot of context about what the user was looking at, or what they would be able to do. New users would sometimes not realize that they were already inside an application because there were no titles, descriptions or clear calls to action.

Screenshot of Contour's zero state, September 2017.

My original intent was simply to update Contour’s entry point, so I wanted to see how other applications were solving the problem. However, I quickly realized that almost all other applications suffered from the same issues, and there was a very apparent lack of consistency across the suite.

Example of some of the other app's entry points pre-redesign.

So rather than improving Contour’s state alone, I decided that it would be best to design a unified entry point that could work for each application. I started talking to different teams to collect context on their needs, as well as determine any outstanding requirements or limitations. From here I developed a few “How Might We” questions to help me orient the problem space.

Goals

  1. How might we make it clear what application a user is currently in and what this application does?

  2. How might we make it easier for users to jump into their workflows?

  3. How might we unify the Foundry applications into a cohesive ecosystem?

Process

While gathering context from stakeholders and trying to align on feature priorities, I started looking at other products to understand what works well and what doesn’t.

From there I drew some low fidelity wireframes that helped facilitate conversations with different teams and made it clear to leadership what the value was that I wanted to provide.

Creating a comparative analysis of various zero states.
Mapping out all the different ways users can access an application.
Brainstorming welcome screen layout.

An essential priority was designing something that would work for all applications regardless of that app’s own complexity or limitations.

Once I felt comfortable with the basic framework for the zero states, I moved into high fidelity. I wanted to make sure that the zero states were not only useful, but also aesthetically pleasing so that onboarding would feel less intimidating to new users.

Final design

The effort to improve the Welcome Screens resulted in introducing several features to the default state of each app. Files were reorganized into recents and favorites to increase discoverability, documentation was given a more prominent position within the app, and a visual branding system was developed to tie all of the applications within Foundry together.

Initial state

To help users orient themselves when they first land in an app, we made the zero state the same across all applications. This way relevant information could always be found in the same place.

Tabs

The new tab menu makes it easy for uses to return to recent workflows or rediscover forgotten files.

Resources

The persistent Help button elevates relevant documentation that each client can configure to suit their own needs.

Visual consistency and illustrations

To ensure that applications would be clearly identifiable within the new default state, I designed a color scheme and brand illustration for each app.

Response

While we did not track quantitative results of the new splash screens, we received very strong, positive verbal feedback from both internal teams and clients. We also observed new users onboarding much faster and more efficiently during user shadowing sessions.

The branding I created for was also adopted for other parts of the Foundry platform. The colors and icons became an integral part of how the apps are distinguised, which helped create a more cohesive brand presence across the platform.
